# Project D 분야 분담과 책임 범위

이 문서는 Project D를 여러 세션과 여러 에이전트가 같은 목표로 나누어 작업하기 위한 분야 분담표다.

## 현재 우선 목표

현재 최우선 목표는 첫 15분 수직 슬라이스다.

목표 루프:

```text
오두막에서 준비
작은 숲으로 원정
수색/루팅/약한 위협
귀환
전리품 분류
제작/퀘스트/거점 갱신
다음 원정 목표 발생
```

## 1. 게임 디렉션 / 기획

책임:

- 핵심 루프 정의
- 첫 30분 흐름
- 용어 정리
- 시스템 우선순위 결정
- 지금 만들 것과 나중에 만들 것 구분

현재 결정:

- 첫 목표는 전체 게임이 아니라 15분짜리 플레이 가능 루프다.
- `레이드`보다 `원정`, `익스트랙션`보다 `귀환`을 세계관 용어로 우선 사용한다.

## 2. 서사 / 세계관 / 퀘스트

책임:

- 디아나의 마력 상실 프롤로그
- 움브라데 추방과 오두막 정착
- 오염된 시체 발견
- 첫 조사 과제
- 세계관 정보의 점진적 공개

현재 결정:

- 마법 폭주는 긴 컷씬보다 짧은 컷씬과 간단한 조작 실패로 표현한다.
- 디아나는 마법을 잃었지만 지식은 남아 있는 학자로 보여준다.
- 첫 원정의 동기는 단순 호기심이 아니라 움브라데를 지키기 위한 원인 조사다.

## 3. 플레이어 / 카메라 / 조작

책임:

- 3인칭 이동
- 카메라
- 상호작용
- 기본 공격
- 회피/달리기
- 입력 설정

첫 수직 슬라이스 범위:

- 이동, 카메라, 상호작용
- 기본 공격 1종
- 회복 아이템 사용
- 간단한 달리기 또는 회피

나중으로 미룸:

- 고급 전투 콤보
- 복잡한 타겟팅
- 다중 무기 전환 연출

## 4. 원정 / 귀환 / 정산

책임:

- 원정 시작
- 원정 준비 UI
- 귀환 지점
- 원정 성공/실패
- 전리품 확정/손실 규칙
- 귀환 정산

첫 수직 슬라이스 범위:

- 오두막에서 첫 원정 시작
- 명확한 귀환 지점 1개
- 귀환 시 루팅품 확정
- 실패 시 일부 손실 또는 치료 자원 소모

나중으로 미룸:

- 다중 조건부 귀환로
- 원정 시간대 선택
- 복잡한 보험/회수 시스템

## 5. 아이템 / 인벤토리 / 루팅

책임:

- 아이템 데이터
- 루팅 컨테이너
- 가방/인벤토리
- 아이템 획득과 분류
- 오두막 창고

첫 수직 슬라이스 범위:

- 아이템 20종 이하
- 루팅 컨테이너 5종 이하
- 단순 가방/창고
- 전리품 분류 화면

나중으로 미룸:

- 복잡한 그리드 회전
- 세밀한 무게/부피/오염 계산
- 대규모 상인 경제

## 6. 제작 / 거점 / 오두막

책임:

- 오두막 상태
- 작업대
- 간단 제작
- 수리
- 보관함
- 거점 업그레이드

첫 수직 슬라이스 범위:

- 재료 제출로 오두막 상태 1단계 개선
- 레시피 3개 이하
- 작업대 1개
- 귀환 후 제작 선택 1개

나중으로 미룸:

- 자유 건축
- 다층 거점 확장 트리
- 복잡한 제작 시간/인력/연료 시스템

## 7. 전투 / 적 / 생존

책임:

- 약한 감염체 AI
- 체력
- 출혈
- 부패 오염
- 회복품
- 기본 전투 피드백

첫 수직 슬라이스 범위:

- 약한 감염체 1종
- 감염 동물은 흔적 또는 단순 이벤트로 처리 가능
- 출혈 또는 부패 오염 중 하나만 확실히 학습
- 전투는 피하거나 처리할 수 있게 설계

나중으로 미룸:

- 인간형 무법자 전투 AI
- 보스
- 복잡한 신체 부위 피해
- 정교한 동물 생태계

## 8. 맵 / 레벨 / 환경 연출

책임:

- 오두막 주변 맵
- 채집 숲길
- 개울가
- 부패 시체 발견지
- 감시초소
- 마법 지뢰 경고 구역
- 조명, 안개, 사운드 변화

첫 수직 슬라이스 범위:

- 작은 순환형 숲 맵
- 평상시와 오염 후 분위기 전환
- 오두막 귀환 루트
- 부패 시체 조사 지점
- 선택 보상 하나

나중으로 미룸:

- 대형 오픈월드
- 여러 지역 이동
- 복잡한 날씨 시스템

## 9. UI / UX

책임:

- 메인 메뉴
- 환경설정
- HUD
- 목표 표시
- 상호작용 표시
- 인벤토리
- 원정 준비
- 귀환 정산

첫 수직 슬라이스 범위:

- 메인 메뉴 기본형
- 그래픽/사운드/조작 최소 설정
- 현재 목표 표시
- 상호작용 하이라이트
- 원정 준비 체크리스트
- 귀환 정산 화면

규칙:

- UI는 Unity Canvas 우선으로 구현한다.

## 10. 기술 기반 / 저장 / Git

책임:

- Unity 6 프로젝트 구조
- 폴더 구조
- ScriptableObject 데이터 구조
- 씬 관리
- 저장/불러오기
- Git 브랜치 관리
- 빌드 설정

첫 수직 슬라이스 범위:

- Unity 6 기본 프로젝트 구조
- 데이터 폴더와 스크립트 폴더 정리
- 최소 저장/불러오기
- 씬 전환 또는 상태 전환
- GitHub push 가능한 상태 유지

나중으로 미룸:

- 고급 세이브 슬롯
- 모딩 구조
- 빌드 자동화

1. 플레이어/카메라/상호작용
2. 아이템/루팅/인벤토리
3. 원정/귀환/정산
4. 오두막/제작/거점
5. 첫 감염체/전투/오염
6. 첫 숲 맵/환경 전환
7. UI/HUD/메뉴
8. 서사/퀘스트 연결
