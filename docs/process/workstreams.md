# Project D 분야 분담과 책임 범위

이 문서는 Project D를 여러 세션과 Codex 서브 에이전트가 같은 목표로 나누어 작업하기 위한 분야 분담표다.

## 현재 우선 목표

현재 최우선 목표는 첫 20분~30분 수직 슬라이스다.

목표 루프:

```text
오두막에서 준비
작은 숲으로 탐사
수색/전리품 획득/약한 위협
귀환
전리품 분류
제작/의뢰/거점 갱신
다음 탐사 목표 발생
```

## 1. 게임 디렉션 / 기획

책임:

- 핵심 루프 정의
- 첫 30분 흐름
- 용어 정리
- 시스템 우선순위 결정
- 지금 만들 것과 나중에 만들 것 구분

현재 결정:

- 첫 목표는 전체 게임이 아니라 20분~30분짜리 플레이 가능 루프다.
- `레이드`의 Project D 용어는 `탐사`다.
- `익스트랙션`과 `탈출`의 Project D 용어는 `귀환`이다.
- `퀘스트`의 Project D 용어는 `의뢰`다.
- `루팅`은 `수색`, `전리품 획득`으로 표현한다.

## 2. 서사 / 세계관 / 의뢰

책임:

- 디아나의 마력 상실 프롤로그
- 움브라데 추방과 오두막 정착
- 오염된 시체 발견
- 첫 의뢰
- 세계관 정보의 점진적 공개

현재 결정:

- 마법 폭주는 긴 컷씬보다 짧은 컷씬과 간단한 조작 실패로 표현한다.
- 마법 폭주는 미니게임 이후 아카데미에 거대한 마법 폭풍이 발생하는 애니메이션 컷씬으로 표현한다.
- 디아나는 마법을 잃었지만 지식은 남아 있는 학자로 보여준다.
- 첫 탐사의 동기는 단순 호기심이 아니라 움브라데를 지키기 위한 원인 조사다.

## 3. 플레이어 / 카메라 / 조작

책임:

- 3인칭 이동
- 카메라
- 상호작용
- 기본 공격
- 회피/달리기
- 입력 설정

첫 수직 슬라이스 범위:

- 이동, 카메라, 상호작용
- 기본 공격 1종
- 회복 아이템 사용
- 간단한 달리기 또는 회피

나중으로 미룸:

- 고급 전투 콤보
- 복잡한 타겟팅
- 다중 무기 전환 연출

## 4. 탐사 / 귀환 / 정리

책임:

- 탐사 시작
- 탐사 준비 UI
- 귀환 지점
- 탐사 성공/실패
- 전리품 확정/손실 규칙
- 귀환 정산

첫 수직 슬라이스 범위:

- 오두막에서 첫 탐사 시작
- 명확한 귀환 지점 1개
- 귀환 시 전리품 확정
- 움브라데 숲 25분 제한시간
- 사망/타임오버 시 짐을 내던지고 오두막으로 급하게 귀환
- 실패 시 아이템 유실
- 보험업자 NPC는 초반 의뢰 라인에서 등장하며, 보험 시스템은 완성형 구현을 뒤로 미룬다.

나중으로 미룸:

- 다중 조건부 귀환로
- 탐사 시간대 선택
- 완성형 보험업자 NPC와 마법생물 회수 시스템

## 5. 아이템 / 인벤토리 / 수색

책임:

- 아이템 데이터
- 수색 컨테이너
- 가방/인벤토리
- 아이템 획득과 분류
- 오두막 창고

첫 수직 슬라이스 범위:

- 아이템 50종 이하
- 수색 컨테이너: 나무상자, 창고상자, 시체, 파밍 포인트 내 랜덤 스폰
- 단순 가방/창고
- 전리품 분류 화면

나중으로 미룸:

- 복잡한 그리드 회전
- 세밀한 무게/부피/오염 계산
- 대규모 상인 경제

## 6. 제작 / 거점 / 오두막

책임:

- 오두막 상태
- 작업대
- 간단 제작
- 수리
- 보관함
- 거점 업그레이드

첫 수직 슬라이스 범위:

- 재료 제출로 오두막 상태 1단계 개선
- 제작 레시피: 무기 2, 방어구 8, 의약품 3, 재료 4, 탄약 2
- 작업대 1개
- 귀환 후 제작 선택 1개

나중으로 미룸:

- 자유 건축
- 다층 거점 확장 트리
- 복잡한 제작 시간/인력/연료 시스템

## 7. 전투 / 적 / 생존

책임:

- 약한 감염체 AI
- 체력
- 회복품
- 기본 전투 피드백

첫 수직 슬라이스 범위:

- 약한 감염체 1종
- 감염 동물은 첫 버전에서 시체로만 등장
- 전투는 피하거나 처리할 수 있게 설계

나중으로 미룸:

- 인간형 무법자 전투 AI
- 보스
- 복잡한 신체 부위 피해
- 정교한 동물 생태계

## 8. 맵 / 레벨 / 환경 연출

책임:

- 오두막 주변 맵
- 채집 숲길
- 개울가
- 부패 시체 발견지
- 숲의 봉우리에 있는 오래된 감시탑
- 마법 지뢰 경고 구역
- 조명, 안개, 사운드 변화

첫 수직 슬라이스 범위:

- 작은 순환형 숲 맵
- 평상시와 오염 후 분위기 전환
- 오두막 귀환 루트
- 숲 봉우리의 오래된 감시탑 확인 목표
- 필수 보상만 제공

나중으로 미룸:

- 대형 오픈월드
- 여러 지역 이동
- 복잡한 날씨 시스템

## 9. UI / UX

책임:

- 메인 메뉴
- 환경설정
- HUD
- 목표 표시
- 상호작용 표시
- 인벤토리
- 탐사 준비
- 귀환 정산

첫 수직 슬라이스 범위:

- 메인 메뉴 기본형
- 그래픽/사운드/조작 최소 설정
- 현재 목표 표시
- 상호작용 하이라이트
- 탐사 준비 체크리스트
- 귀환 정산 화면

규칙:

- UI는 Unity Canvas 우선으로 구현한다.

## 10. 기술 기반 / 저장 / Git

책임:

- Unity 6 프로젝트 구조
- 폴더 구조
- ScriptableObject 데이터 구조
- 씬 관리
- 단일 저장 파일 정책
- Git 브랜치 관리
- 빌드 설정

첫 수직 슬라이스 범위:

- Unity 6 기본 프로젝트 구조
- 데이터 폴더와 스크립트 폴더 정리
- 저장 파일 1개만 사용하는 자동 저장/자동 로드
- 오두막 내부 실시간 저장
- 탐사 종료 후 저장
- 씬 전환 또는 상태 전환
- GitHub push 가능한 상태 유지

나중으로 미룸:

- 별도 세이브 슬롯
- 모딩 구조
- 빌드 자동화
