# Project D 프로젝트 방향과 레퍼런스

## 핵심 방향

Project D는 PC 대상 Unity 6 기반 3D 싱글 플레이어 게임이다.

장르는 판타지, 좀비 아포칼립스, 생존, 익스트랙션 루팅을 결합한 3인칭 RPG로 잡는다.

플레이어는 제한된 자원과 위험한 환경 속에서 탐사, 루팅, 전투, 생존, 탈출을 반복하며 성장한다.

## 개발 기준

- 플랫폼: PC
- 엔진: Unity 6
- 그래픽: 3D
- 플레이 방식: 싱글 플레이어
- 시점: 3인칭
- 캐릭터 방향: 서브컬쳐풍 캐릭터 디자인
- 장르 축: 판타지 + 좀비 아포칼립스 + 생존 + 익스트랙션 루팅
- UI 구현 기준: Unity Canvas 우선
- 이미지 생성 기준: imagegen 스킬 사용

## 레퍼런스

### 그래픽 / 캐릭터 톤

- 젠레스 존 제로
- 명조
- 명일방주: 엔드필드

의도:

- 실사풍보다는 선명한 스타일라이즈드 3D를 지향한다.
- 캐릭터는 서브컬쳐풍 매력을 유지하되, 세계관은 어둡고 생존 압박이 느껴지게 만든다.
- 판타지 배경이라도 지나치게 고전적인 중세풍만으로 고정하지 않고, 게임적 가독성과 캐릭터성을 우선한다.

### 게임 장르 / 루프

- 이스케이프 프롬 타르코프: 익스트랙션 루팅 구조
- 데이즈: 좀비 아포칼립스 생존감

의도:

- 맵에 진입한다.
- 위험을 감수하고 탐색, 전투, 루팅을 한다.
- 확보한 자원과 정보를 들고 탈출한다.
- 탈출 결과로 거점, 장비, 제작, 연구, 생존 기반을 강화한다.

싱글 플레이어 게임이므로 PvP 긴장감 대신 다음 요소로 압박을 만든다.

- 제한된 자원
- 좀비 감염과 부패한 마력
- 지뢰, 함정, 오염지대
- 적대 세력 또는 정찰대
- 시간, 소음, 부상, 무게, 탈출 지점 조건

### 배경 / 맵 톤

- 리그 오브 레전드: 데마시아
- 이스케이프 프롬 타르코프: 모든 맵

의도:

- 데마시아식의 질서정연한 판타지 왕국, 성채, 석조 건축, 군사적 상징성을 참고한다.
- 타르코프식의 밀도 높은 탐색 공간, 폐허, 루팅 포인트, 위험 구역, 탈출 동선을 참고한다.
- 판타지 지역도 단순한 배경이 아니라 루팅, 위험, 동선, 은폐, 전투 판단이 살아있는 플레이 공간으로 만든다.

## 역할 분담

사용자가 직접 담당 가능한 영역:

- 캐릭터 디자인
- 캐릭터 모델링
- 캐릭터 애니메이션

Codex가 적극적으로 도울 영역:

- Unity 6 프로젝트 구조 설계
- 3인칭 플레이어 컨트롤러
- 카메라, 조준, 상호작용 시스템
- 인벤토리, 장비, 루팅 시스템
- 익스트랙션 진입/탈출 루프
- 좀비 AI, 감염, 부패한 마력 시스템
- 생존 요소: 체력, 부상, 스태미나, 허기, 갈증, 무게
- 제작, 거점, 연구, 업그레이드 구조
- 퀘스트와 목표 시스템
- 맵 이벤트, 스폰, 위험 구역, 탈출 지점
- Unity Canvas 기반 HUD와 메뉴 UI
- 저장/불러오기
- 데이터 테이블, ScriptableObject 기반 밸런싱 구조
- 개발용 디버그 도구와 테스트 씬
- GitHub 기반 작업 동기화와 브랜치 관리

## 초반 개발 우선순위

1. 프로젝트 규칙과 문서 구조 고정
2. Unity 6 빈 프로젝트 생성 및 Git 관리 기준 정리
3. 3인칭 이동, 카메라, 기본 상호작용 프로토타입
4. 루팅 가능한 컨테이너와 아이템 데이터 구조
5. 인벤토리와 장비 슬롯
6. 테스트 맵 1개와 탈출 지점
7. 단순 좀비 AI와 전투/피격 루프
8. 탐사 시작, 루팅, 탈출, 결과 정산으로 이어지는 최소 게임 루프

## 디자인 메모

이 게임은 "귀여운 캐릭터가 나오는 좀비 게임"이 아니라, 서브컬쳐풍 캐릭터성과 어두운 생존 익스트랙션 구조가 공존하는 방향을 목표로 한다.

디아나는 마력을 잃은 엘프이므로, 전형적인 마법사 플레이보다 도구, 제작, 지형 이해, 준비, 회피, 제한적인 전투를 통해 살아남는 캐릭터로 설계하기 좋다.

움브라데는 단순한 숲이 아니라 전쟁의 상흔, 마법 지뢰, 부패한 마력, 자연 회복, 외부 세력의 위협이 겹친 핵심 플레이 공간으로 잡는다.
