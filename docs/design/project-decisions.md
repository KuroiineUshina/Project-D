# Project D 프로젝트 결정표

이 문서는 사용자가 직접 확정한 Project D의 주요 결정 사항을 기록한다. Codex와 Codex 서브 에이전트는 이 문서를 기준으로 작업한다.

## A. 용어

- `레이드`의 Project D 용어: `탐사`
- `익스트랙션`, `탈출`의 Project D 용어: `귀환`
- `퀘스트`의 Project D 용어: `의뢰`
- `루팅`의 Project D 용어: `수색`, `전리품 획득`

## B. 제한시간

- 제한시간 원인: 감염 원인이 되는 바이러스가 공기 중에 퍼져 있음
- 움브라데 숲 제한시간: 25분
- 타이머 표시 방식:
  - `O`를 누르면 타르코프처럼 타이머가 등장한다.
  - `O`를 더블 탭하면 귀환 지점 목록까지 표시한다.
- 인벤토리/메뉴 중 타이머: 없음
- 타이머 시작 시점: 실제 탐사 지역에서 플레이어 조작이 가능한 시점

## C. 실패 처리

- 실패 시 잃는 짐 범위: 타르코프 기준 secure container 내부 물건을 제외한 전부와 동일한 구조
- Project D의 secure container 대응 명칭: 미정
- 기본 무기/기본 도구 손실 여부: 전부 손실 가능
- 의뢰 핵심 아이템 손실 여부: 전부 손실 가능
- 실패 후 페널티: 아이템 유실
- 첫 튜토리얼 탐사 페널티: 아이템 유실

## D. 보험 시스템

- 보험업자 NPC 등장 시점: 초반 의뢰 라인에서 등장
- 보험 가입 방식:
  - 보험업자에게 직접 가서 보험을 들 수 있다.
  - 탐사 전 보험드는 단계가 있다.
- 보험 대상: 소모품이 아닌 모든 것
- 마법생물 하수인 형태: 미정
- 회수 성공률:
  - 기본 100%
  - 일부 맵에서는 확률 감소 또는 회수 불가

## E. 메인 메뉴 / 저장

- 메인 메뉴 항목:
  - 게임시작
  - 환경설정
  - 게임종료
- 시작 버튼 동작: 바로 게임을 시작하고 세이브 파일을 불러온다.
- 자동 저장 시점:
  - 오두막 내부에서는 실시간 저장
  - 탐사 중에는 저장하지 않고, 탐사가 끝난 뒤 저장
- 저장 파일 삭제/초기화 안내:
  - `환경설정 -> 기타 -> 세계 초기화`

## F. 첫 탐사 구성

- 무법자 등장 방식: 도망가다 감염체에게 당하는 연출로 등장
- 감염 동물 등장 방식: 시체로만 등장
- 첫 적 구성: 감염체
- 첫 탐사 목표: 숲의 봉우리에 있는 오래된 감시탑에 올라 주변 확인하기
- 선택 보상 여부: 필수 보상만 있음

## G. 오두막 / 튜토리얼

- 오두막 수리 방식: 창고에 있는 재료를 이용하여 상호작용
- 첫 튜토리얼 분량: 20분~30분
- 오두막 이름:
  - 초반에는 `오두막`으로 부른다.
  - 규모가 커지는 스토리 중반에 마을 형태로 변한다.
  - 마을 이름은 플레이어가 짓는다.
- 연구 책상/망가진 마도구 배치 여부: 완전히 버려진 곳이므로 없음

## H. 프롤로그

- 마법 폭주 표현 방식: 미니게임 이후 거대한 마법 폭풍이 아카데미에서 발생
- 연구실 프롤로그 구현 방식: 미니게임과 애니메이션 컷씬 재생
- 추방 과정 표현 방식: 애니메이션 컷씬

## I. 첫 수직 슬라이스 범위

- 첫 목표: 준비 -> 탐사 -> 복귀 -> 정리 -> 제작 사이클 반복
- 초기 아이템 수: 무기/방어구/의약품/재료/소모품 등 총 50개 이하
- 초기 제작 레시피:
  - 무기 2
  - 방어구 8
  - 의약품 3
  - 재료 4
  - 탄약 2
- 초기 수색 컨테이너:
  - 나무상자
  - 창고상자
  - 시체
  - 파밍 포인트 내 랜덤 스폰
- 첫 버전 제외 시스템: 미정

## J. Codex 서브 에이전트 작업 규칙

- 평가/조사 서브 에이전트 파일 수정 금지: 확정
- 구현/문서 서브 에이전트만 파일 수정 가능: 확정
- 서브 에이전트 결과 최종 반영 방식: Codex가 최종 검토 후 반영

## 사용자 직접 작성 예정

사용자가 직접 작성할 영역:

- 스토리
- 의뢰 라인
- 아이디어 문서에 없는 추가 지역 설정
- 아이디어 문서에 없는 추가 NPC 설정
- 캐릭터 디자인, 모델링, 애니메이션

Codex 역할:

- 사용자가 구축한 세계관에 대한 질문
- 설정 모순, 게임화 가능성, 구현 범위에 대한 피드백
- 문서 구조화
- Unity 6 구현 지원
- GitHub 작업 동기화

## 이미지 생성 메모

캐릭터 시트 등 이미지가 필요할 가능성이 있다. 이미지는 반드시 imagegen 스킬을 사용해서 생성한다.

- 마법생물 하수인 형태
- 첫 버전 제외 시스템 목록
